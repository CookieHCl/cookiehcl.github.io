Block Ciphers

Posted on 2025-09-09 In SNU , 4-2 , 현대암호학 개론 Reading time ≈ 7 mins.

Motivation

OTP, Stream Ciphers only assumes ciphertext-only attack.
We want to securely send multiple ciphertexts using same key!

Block ciphers are used to build CPA-secure encryption schemes.
Block ciphers are not encryption schemes!
But it can used as encryptions, MACs(Message Authentication Code), hash functions, PRGs, ...

PRFs and PRPs

A function E:X→XE: X \rightarrow XE:X→X is a permutation if there exists and inverse function such that ∀x,E−1(E(x))=x\forall x, E^{-1}(E(x)) = x∀x,E−1(E(x))=x.

A pseudorandom function (PRF) is a keyed function F:K×X→YF: K \times X \rightarrow YF:K×X→Y that is efficiently computable by a deterministic algorithm.
Usually K={0,1}n,X={0,1}lin,Y={0,1}loutK = \{0,1\}^n, X = \{0,1\}^{l_{in}}, Y = \{0,1\}^{l_{out}}K={0,1}n,X={0,1}lin​,Y={0,1}lout​, and we can get a function Fk:X→YF_k: X \rightarrow YFk​:X→Y for each k∈Kk \in Kk∈K.

A pseudorandom permutation (PRP) is a keyed function E:K×X→XE: K \times X \rightarrow XE:K×X→X such that for all k∈Kk \in Kk∈K, Ek(x)=E(k,x)E_k(x) = E(k, x)Ek​(x)=E(k,x) is a permutation, and Ek,Ek−1E_k, E_k^{-1}Ek​,Ek−1​ are efficiently computable by a deterministic algorithm.
Usually K={0,1}n,X={0,1}lK = \{0,1\}^n, X = \{0,1\}^lK={0,1}n,X={0,1}l, and we can get a permutation Ek:X→XE_k: X \rightarrow XEk​:X→X for each k∈Kk \in Kk∈K.

Basically PRF and PRP is a function/permutation version of PRG!
Note that this is way harder than PRG; We can only have ∣K∣|K|∣K∣ functions, but there are ∣Y∣∣X∣|Y|^{|X|}∣Y∣∣X∣ distinct functions and ∣X∣!|X|!∣X∣! distinct permutations!

Secure PRFs

Let F\mathcal{F}F is the set of all functions from X to Y.
FkF_kFk​ (for uniform k∈Kk \in Kk∈K) should be indistinguishable from fff (for uniform f∈Ff \in \mathcal{F}f∈F)!

Problem: We can't give local table of function fff, it's too large!
Instead, adversary can query up to qqq times.

Let F:K×X→YF: K \times X \rightarrow YF:K×X→Y be a PRF. For a distinguisher A\mathcal{A}A, its advantage is defined as

AdvF(A)≔∣Pr⁡[EXP0,F(A)=1]−Pr⁡[EXP1,F(A)=1]∣Adv_F(\mathcal{A}) \coloneqq \left| \Pr[EXP_{0,F}(\mathcal{A}) = 1] - \Pr[EXP_{1,F}(\mathcal{A}) = 1] \right| AdvF​(A):=∣Pr[EXP0,F​(A)=1]−Pr[EXP1,F​(A)=1]∣

We say that F is secure if AdvF(A)Adv_F(\mathcal{A})AdvF​(A) is negligible for any efficient A\mathcal{A}A.

We simply denote EXP0,F(A)=AFk(⋅),EXP1,F(A)=Af(⋅)EXP_{0,F}(\mathcal{A}) = \mathcal{A}^{F_k(\cdot)}, EXP_{1,F}(\mathcal{A}) = \mathcal{A}^{f(\cdot)}EXP0,F​(A)=AFk​(⋅),EXP1,F​(A)=Af(⋅).
cf. Af(⋅)\mathcal{A}^{f(\cdot)}Af(⋅) actually means that adversary has a oracle access to fff, i.e. adversary can use fff freely (up to qqq times).

Secure PRPs

Let P\mathcal{P}P is the set of all permutations of X.
EkE_kEk​ (for uniform k∈Kk \in Kk∈K) should be indistinguishable from fff (for uniform f∈Pf \in \mathcal{P}f∈P)!

Let E:K×X→XE: K \times X \rightarrow XE:K×X→X be a PRP. For a distinguisher A\mathcal{A}A, its advantage is defined as

AdvE(A)≔∣Pr⁡[EXP0,E(A)=1]−Pr⁡[EXP1,E(A)=1]∣Adv_E(\mathcal{A}) \coloneqq \left| \Pr[EXP_{0,E}(\mathcal{A}) = 1] - \Pr[EXP_{1,E}(\mathcal{A}) = 1] \right| AdvE​(A):=∣Pr[EXP0,E​(A)=1]−Pr[EXP1,E​(A)=1]∣

We say that E is secure if AdvE(A)Adv_E(\mathcal{A})AdvE​(A) is negligible for any efficient A\mathcal{A}A.

We simply denote EXP0,E(A)=AEk(⋅),EXP1,E(A)=Af(⋅)EXP_{0,E}(\mathcal{A}) = \mathcal{A}^{E_k(\cdot)}, EXP_{1,E}(\mathcal{A}) = \mathcal{A}^{f(\cdot)}EXP0,E​(A)=AEk​(⋅),EXP1,E​(A)=Af(⋅).

PRG from PRF

We can make secure PRG from secure PRF!

Let F:{0,1}n×{0,1}n→{0,1}nF: \{0, 1\}^n \times \{0, 1\}^n \rightarrow \{0, 1\}^nF:{0,1}n×{0,1}n→{0,1}n be a secure PRF. Then, the following G:{0,1}n→{0,1}nmG: \{0, 1\}^n \rightarrow \{0, 1\}^{nm}G:{0,1}n→{0,1}nm is a secure PRG.

G(k)≔Fk(0)∥Fk(1)∥⋯∥Fk(m−1)G(k) \coloneqq F_k(0) \Vert F_k(1) \Vert \cdots \Vert F_k(m-1) G(k):=Fk​(0)∥Fk​(1)∥⋯∥Fk​(m−1)

Proof

Let's assume that GGG is not secure, i.e. there exists an adversary A\mathcal{A}A and non-negligible ϵ>0\epsilon > 0ϵ>0 such that

∣Pr⁡k[A(G(k))=1]−Pr⁡r[A(r)=1]∣>ϵ.\left| \Pr_k[ \mathcal{A}(G(k)) = 1 ] - \Pr_r[ \mathcal{A}(r) = 1 ] \right| > \epsilon. ​kPr​[A(G(k))=1]−rPr​[A(r)=1]​>ϵ.

We define an adversary B\mathcal{B}B for PRF experiment.

  1. Query mmm times and get yiy_iyi​ (Fk(i)F_k(i)Fk​(i) or f(i)f(i)f(i)) for 0≤i≤m−10 \leq i \leq m-10≤i≤m−1.
  2. Create string ω≔y0∥y1∥⋯∥ym−1\omega \coloneqq y_0 \Vert y_1 \Vert \cdots \Vert y_{m-1}ω:=y0​∥y1​∥⋯∥ym−1​ by appending yiy_iyi​.
  3. Perform secure PRG experiment and get the output b′≔A(ω)b' \coloneqq \mathcal{A}(\omega)b′:=A(ω).
  4. Output b′b'b′ directly.

We'll prove that B\mathcal{B}B can distinguish PRF with a non-negligible probability.

If EXP0EXP_0EXP0​ (i.e. yi=Fk(i)y_i = F_k(i)yi​=Fk​(i)), ω\omegaω has the same distribution as G(k)G(k)G(k). ∴,Pr⁡[EXP0,F(B)=1]=Pr⁡k[A(G(k))=1]\therefore, \Pr[EXP_{0,F}(\mathcal{B}) = 1] = \Pr_k[\mathcal{A}(G(k)) = 1]∴,Pr[EXP0,F​(B)=1]=Prk​[A(G(k))=1].
If EXP1EXP_1EXP1​ (i.e. yi=f(i)y_i = f(i)yi​=f(i)), ω\omegaω is uniformly random string. ∴,Pr⁡[EXP1,F(B)=1]=Pr⁡r[A(r)=1]\therefore, \Pr[EXP_{1,F}(\mathcal{B}) = 1] = \Pr_r[\mathcal{A}(r) = 1]∴,Pr[EXP1,F​(B)=1]=Prr​[A(r)=1].

∴AdvF(B)=∣Pr⁡[EXP0,F(B)=1]−Pr⁡[EXP1,F(B)=1]∣=∣Pr⁡k[A(G(k))=1]−Pr⁡r[A(r)=1]∣>ϵ>0\begin{align*} \therefore Adv_F(\mathcal{B}) &= |\Pr[EXP_{0,F}(\mathcal{B}) = 1] - \Pr[EXP_{1,F}(\mathcal{B}) = 1]| \\ &= \left| \Pr_k[ \mathcal{A}(G(k)) = 1 ] - \Pr_r[ \mathcal{A}(r) = 1 ] \right| \\ &> \epsilon > 0 \end{align*}∴AdvF​(B)​=∣Pr[EXP0,F​(B)=1]−Pr[EXP1,F​(B)=1]∣=​kPr​[A(G(k))=1]−rPr​[A(r)=1]​>ϵ>0​

Since F is not secure when G is not secure, G is secure when F is secure!

Note that this PRG is also parallelizable.

PRF switching lemma (PRF from PRP)

Any secure PRP is also a secure PRF, if |X| is sufficiently large.

Let E:{0,1}n×{0,1}n→{0,1}nE: \{0, 1\}^n \times \{0, 1\}^n \rightarrow \{0, 1\}^nE:{0,1}n×{0,1}n→{0,1}n be a PRP. Then, for any qqq-query adversary A\mathcal{A}A:

∣AdvEPRF(A)−AdvEPRP(A)∣≤q22∣X∣\left| Adv_E^{PRF}(\mathcal{A}) - Adv_E^{PRP}(\mathcal{A}) \right| \leq \frac{q^2}{2|X|} ​AdvEPRF​(A)−AdvEPRP​(A)​≤2∣X∣q2​

c.f. q22∣X∣\frac{q^2}{2|X|}2∣X∣q2​ comes from the birthday paradox.
Let's choose qqq values from XXX uniformly at random thereby allowing repetitions.
The probability of at least one value is chosen more than once is less than q22∣X∣\frac {q^2}{2|X|}2∣X∣q2​ (also called the birthday bound).

Proof of the birthday bound

Let's say the i-th value is xix_ixi​.

Pr⁡[∃i<j:xi=xj]≤∑1≤i<j≤qPr⁡[xi=xj]=(q2)⋅1∣X∣<q22∣X∣\begin{align*} \Pr[\exists i < j : x_i = x_j] &\leq \sum_{1 \leq i < j \leq q} \Pr[x_i = x_j] \\ &= \binom{q}{2} \cdot \frac{1}{|X|} < \frac{q^2}{2|X|} \end{align*}Pr[∃i<j:xi​=xj​]​≤1≤i<j≤q∑​Pr[xi​=xj​]=(2q​)⋅∣X∣1​<2∣X∣q2​​

Proof of the switching lemma

Let fff is a random PRF and ggg is a random PRP. From the definition,

AdvEPRF(A)=∣Pr⁡[AEk(⋅)=1]−Pr⁡[Af(⋅)=1]∣AdvEPRP(A)=∣Pr⁡[AEk(⋅)=1]−Pr⁡[Ag(⋅)=1]∣\begin{align*} Adv_E^{PRF}(\mathcal{A}) &= \left| \Pr[\mathcal{A}^{E_k(\cdot)} = 1] - \Pr[\mathcal{A}^{f(\cdot)} = 1] \right| \\ Adv_E^{PRP}(\mathcal{A}) &= \left| \Pr[\mathcal{A}^{E_k(\cdot)} = 1] - \Pr[\mathcal{A}^{g(\cdot)} = 1] \right| \end{align*}AdvEPRF​(A)AdvEPRP​(A)​=​Pr[AEk​(⋅)=1]−Pr[Af(⋅)=1]​=​Pr[AEk​(⋅)=1]−Pr[Ag(⋅)=1]​​

Therefore, from the reverse triangular inequality ∣∣a∣−∣b∣∣≤∣a−b∣||a|-|b|| \leq |a-b|∣∣a∣−∣b∣∣≤∣a−b∣,

∣AdvEPRF(A)−AdvEPRP(A)∣≤∣Pr⁡[Ag(⋅)=1]−Pr⁡[Af(⋅)=1]∣\begin{align*} \left| Adv_E^{PRF}(\mathcal{A}) - Adv_E^{PRP}(\mathcal{A}) \right| \leq \left| \Pr[\mathcal{A}^{g(\cdot)} = 1] - \Pr[\mathcal{A}^{f(\cdot)} = 1] \right| \end{align*}​AdvEPRF​(A)−AdvEPRP​(A)​≤​Pr[Ag(⋅)=1]−Pr[Af(⋅)=1]​​

Now we prove the difference lemma: Let Z,W0,W1Z, W_0, W_1Z,W0​,W1​ be the events defined over some probability space, and W0∧¬ZW_0 \land \lnot ZW0​∧¬Z occurs if and only if W1∧¬ZW_1 \land \lnot ZW1​∧¬Z. (i.e. Pr⁡[W0∧¬Z]=Pr⁡[W1∧¬Z]\Pr[W_0 \land \lnot Z] = \Pr[W_1 \land \lnot Z]Pr[W0​∧¬Z]=Pr[W1​∧¬Z]) Then the following holds.

∣Pr⁡[W0]−Pr⁡[W1]=∣(Pr⁡[W0∧Z]+Pr⁡[W0∧¬Z])−(Pr⁡[W1∧Z]+Pr⁡[W1∧¬Z])=∣Pr⁡[W0∧Z]−Pr⁡[W1∧Z]∣≤Pr⁡[Z](∵0≤Pr⁡[W0∧Z],Pr⁡[W1∧Z]≤Pr⁡[Z])\begin{align*} |\Pr[W_0] - \Pr[W_1] &= |(\Pr[W_0 \land Z] + \Pr[W_0 \land \lnot Z]) - (\Pr[W_1 \land Z] + \Pr[W_1 \land \lnot Z]) \\ &= |\Pr[W_0 \land Z] - \Pr[W_1 \land Z]| \\ &\leq \Pr[Z] \quad (\because 0 \leq \Pr[W_0 \land Z], \Pr[W_1 \land Z] \leq \Pr[Z]) \end{align*}∣Pr[W0​]−Pr[W1​]​=∣(Pr[W0​∧Z]+Pr[W0​∧¬Z])−(Pr[W1​∧Z]+Pr[W1​∧¬Z])=∣Pr[W0​∧Z]−Pr[W1​∧Z]∣≤Pr[Z](∵0≤Pr[W0​∧Z],Pr[W1​∧Z]≤Pr[Z])​

Let x1,…xqx_1, \ldots x_qx1​,…xq​ are the variables that A\mathcal{A}A queried, and ZZZ is an event that at least two of f(xi)f(x_i)f(xi​) have the same value.
If ¬Z\lnot Z¬Z occurs, because f(xi)f(x_i)f(xi​) and g(xi)g(x_i)g(xi​) are random values with no collisions, f(xi)f(x_i)f(xi​) and g(xi)g(x_i)g(xi​) have same distribution.
∴Pr⁡[Ag(⋅)=1∧¬Z]=Pr⁡[Af(⋅)=1∧¬Z]\therefore \Pr[\mathcal{A}^{g(\cdot)} = 1 \land \lnot Z] = \Pr[\mathcal{A}^{f(\cdot)} = 1 \land \lnot Z]∴Pr[Ag(⋅)=1∧¬Z]=Pr[Af(⋅)=1∧¬Z]

Also, from the birthday bound, Pr⁡[Z]<q22∣X∣\Pr[Z] < \frac{q^2}{2|X|}Pr[Z]<2∣X∣q2​.
Therefore, by using difference lemma,

∣AdvEPRF(A)−AdvEPRP(A)∣≤∣Pr⁡[Ag(⋅)=1]−Pr⁡[Af(⋅)=1]∣≤Pr⁡[Z]<q22∣X∣\begin{align*} \left| Adv_E^{PRF}(\mathcal{A}) - Adv_E^{PRP}(\mathcal{A}) \right| &\leq \left| \Pr[\mathcal{A}^{g(\cdot)} = 1] - \Pr[\mathcal{A}^{f(\cdot)} = 1] \right| \\ &\leq \Pr[Z] < \frac{q^2}{2|X|} \end{align*}​AdvEPRF​(A)−AdvEPRP​(A)​​≤​Pr[Ag(⋅)=1]−Pr[Af(⋅)=1]​≤Pr[Z]<2∣X∣q2​​

Lemma: If ∣X∣|X|∣X∣ is sufficiently large so that q22∣X∣\frac{q^2}{2|X|}2∣X∣q2​ is negligible, then AdvEPRP(A)Adv_E^{PRP}(\mathcal{A})AdvEPRP​(A) is negligible implies that AdvEPRF(A)Adv_E^{PRF}(\mathcal{A})AdvEPRF​(A) is negligible.
i.e. Any secure PRP is also a secure PRF, if |X| is sufficiently large!

Recall) We can make secure stream cipher from secure PRG!
Therefore secure PRP can make secure PRF, secure PRF can make secure PRG, and secure PRG can make secure stream cipher!

Block Ciphers

Block ciphers are actually just PRPs with same plaintext/ciphertext size (key size can be different).

  • M=C={0,1}lM = C = \{ 0, 1 \}^lM=C={0,1}l
  • K={0,1}nK = \{ 0, 1 \}^nK={0,1}n

Block cipher generates r keys (k1,…,krk_1, \ldots, k_rk1​,…,kr​) from k∈Kk \in Kk∈K, then compute round function mi=Rnd(ki,mi−1)m_i = Rnd(k_i, m_{i - 1})mi​=Rnd(ki​,mi−1​) (where m0=mm_0 = mm0​=m, mr=cm_r = cmr​=c).

Round function can be anything: lookup table, xor, shift operation, ...

Data Encryption Standard (DES)

In 1972, NIST calls for a block cipher standard.
In 1974, IBM designs Lucifer.
Lucifer was later renamed to DES.

n=56,l=64,r=16n = 56, l = 64, r = 16 n=56,l=64,r=16

It was secure, but it was broken by exhaustive search in 1997.
Later, DES was replaced by AES in 2001.

Advanced Encryption Standard (AES)

AES used the similar method as DES, but with an increased key size.
In fact, IBM proposed 128-bit key, but NSA reduced it to 56-bit key.
There is a conspiracy theory that the key size was reduced to allow the NSA to break it.

n=128/192/256,l=128,r=10/12/14n = 128/192/256, l = 128, r = 10/12/14 n=128/192/256,l=128,r=10/12/14

AES is designed with simple operations, with a mixture of linear and non-linear operation. (Looks like NN...?)
Simple operations were used for faster running time and hardware efficiency. Modern CPU can compute one round of AES in one cycle!!!
One can increase code size (pre-compute) for faster AES, or reduce code size for smaller AES (used in low memory systems).

Attacks on Block Ciphers

DES have some advanced attacks: e.g. differential cryptanalysis, linear cryptanalysis.
AES is designed to resist both attacks, and only brute force attack is known.

There is a best key recovery attack which can crack AES four times better than exhaustive search... But it is necessary? (2128→21262^{128} \rightarrow 2^{126}2128→2126)
Just run with four CPUs...

Grover's algorithm can recover the key using quantum computer in time O(∣K∣12)O \left( |K|^{\frac{1}{2}} \right)O(∣K∣21​)!
However, quantum computer's one operation is very different from standard computer's operation.
Even when quantum computers are invented, it is expected that the running time will be similar.

Strengthening Block Ciphers?

Let E:K×M→ME: K \times M \rightarrow ME:K×M→M be a block cipher.
Can we make better block cipher by using block cipher twice? 2E:K2×M→M2E: K^2 \times M \rightarrow M2E:K2×M→M, c=2Ek1,k2(m)=Ek1(Ek2(m))c = 2E_{k_1, k_2}(m) = E_{k_1}(E_{k_2}(m))c=2Ek1​,k2​​(m)=Ek1​​(Ek2​​(m))
2E is two times slower than E, but key size is squared!

However, this leaks critical information: Dk1(c)=Ek2(m)D_{k_1}(c) = E_{k_2}(m)Dk1​​(c)=Ek2​​(m)
We can use Meet-in-the-Middle (MitM) Attack!

  1. Get a plaintext-ciphertext pair (m, c).
  2. Build the table of (k2,Ek2(m))\left( k_2, E_{k_2}(m) \right)(k2​,Ek2​​(m)) for all k2∈Kk_2 \in Kk2​∈K, and sort on the second column (Ek2(m)E_{k_2}(m)Ek2​​(m)).
  3. For all k1∈Kk_1 \in Kk1​∈K, check if Dk1(c)D_{k_1}(c)Dk1​​(c) is in the table.
  • Space complexity: ∣K∣⋅log⁡∣M∣|K| \cdot \log |M|∣K∣⋅log∣M∣
  • Time complexity: ∣K∣⋅log⁡∣K∣+∣K∣⋅log⁡∣K∣|K| \cdot \log |K| + |K| \cdot \log |K|∣K∣⋅log∣K∣+∣K∣⋅log∣K∣ (building + lookup)

Then how about using block cipher thrice...???
This is actually quite secure!!
Because space constraint is more challenging, we build the table of (k3,Ek3(m))\left( k_3, E_{k_3}(m) \right)(k3​,Ek3​​(m)) and check if Dk2(Dk1(c))D_{k_2}(D_{k_1}(c))Dk2​​(Dk1​​(c)) is in the table.
Time complexity is larger than ∣K∣2|K|^2∣K∣2!

Triple-DES was actually used (and it's standard!) when people were migrating to AES.
You could use existing DES hardware to achieve AES-level security!
