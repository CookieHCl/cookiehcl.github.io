Physical memory를 전부 page 크기로나눔  
Virtual memory는 아주 큰 memory space -> Page table로 이어줌

Virtual memory -> Pages  
Physical memory -> Frames  
둘이 항상 같아야함 (But page 크기가 여러 개인 경우도 있긴 함)

Fixed size + 연속적일 필요 없음 -> 메모리 관리가 훨씬 쉬워짐

Page table entry 수 = 가능한 Virtual page 넘버의 수

page table 주소도 프로세스마다 다르니까 레지스터로 관리

PTE: page table의 하나의 entry  
중요: Valid bit 존재
0 -> 쓰면 안 됨 (이게 왜 있냐? 사실 일종의 cache miss; 아직 안 읽어온 메모리임 or NULL 처럼 진짜 안 되는 page table을 access하는거 -> CPU가 trap 걸어서 OS가 판단하게 함)

Demanding paging: 처음으로 page table에 할당하거나 한동안 안 써서 메모리에 없어진 경우 page table이 정당해도 fault가 생김

paging with segmentation: 안 씀
Multi-level page table: 다음 page table 중 하나라도 있으면 valid bit on
사실 radix tree랑 똑같음  
IA-32: Page table이 전부 4KB
